# QuizApp

## Курсова робота на тему: "Вікторина"

# Про проект

QuizApp — це десктопний додаток, написаний на C# / WPF, який дозволяє:

- проходити тести за обраними категоріями
- додавати/редагувати питання та варіанти відповідей
- керувати категоріями
- зберігати результати тестування користувача
- переглядати історію спроб
- працювати з системою користувачів (вхід, реєстрація)

# Локальний запуск

1. Завантажити репозиторій

   ```bash
   git clone https://github.com/NaotoAzazel/QuizApp.git
   cd QuizApp
   ```

2. [Налаштувати](#налаштування-конфігураційного-файлу) конфігураціонний файл

# Функціональність

1. Користувач

- Реєстрація з логіном, паролем та датою народження (неможливо зареєструвати вже існуючий логін).
- Авторизація за логіном і паролем.
- Після входу користувач може:
  - стартувати нову вікторину;
  - переглянути результати своїх минулих вікторин;
  - переглянути Топ-20 результатів за конкретною вікториною;
  - змінити налаштування: пароль та дату народження;

2. Категорії вікторин

- Вибір категорії для проходження вікторини (наприклад, «Історія», «Географія», «Біологія»).
- Підтримка «Mixed»-категорії (змішана вікторина з питань з різних категорій, що обираються випадково).
- Додавання, редагування та видалення категорій (для адміністратора).
- Позначення видалених категорій у UI.
- Фільтрація питань за категорією.

3. Питання

- Створення, редагування та видалення питань (для адміністратора).
- Додавання варіантів відповідей до питання.
- Вибір правильного варіанту/варіантів відповідей.
- Питання можуть мати один або декілька правильних варіантів: якщо користувач не відповів на всі правильні варіанти, питання не зараховується.

4. Проходження вікторини

- Випадковий порядок питань під час проходження.
- Кожна вікторина складається з 20 питань.
- Підрахунок правильних відповідей користувача.
- Збереження результату у базі даних.
- Показ кількості правильних відповідей та місця користувача у таблиці результатів.

5. Адміністратор

- Вхід у панель адміністратора.
- Можливість керувати категоріями та питаннями.

# Технології

- C# (.NET) - основна логіка
- WPF - UI
- SQLite - вбудована БД
- Entity Framework Core - ORM
- Repository Pattern - абстракція доступу до даних
- LINQ - запити до БД
- XAML - розмітка UI

# Налаштування конфігураційного файлу

> [!WARNING]

> Для запуску додатку в різних режимах (dev/prod) створіть файл "appsettings.[Development/Production].json"

### База даних

Почнемо з налаштування нашої бази даних. Ви можете використовувати будь-яку Postgres. Отримайте змінні з вашого середовища бази даних і вставте їх у `ConnectionStrings -> db`

### Адмін

Вставте ім'я адмінистратора в полі `AdminSettings -> username`.

> [!NOTE]
> Ім'я використовується для відображення кнопки, яка веде до адмін-панелі.

> [!CAUTION]
> Якщо вказати ім’я неправильно, доступ до адмін-панелі можуть отримати сторонні користувачі.
